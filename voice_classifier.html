Speaker Classification Policy Evolution

V1 — Host-Only Classifier

WhisperX segmentation + Pyannote embeddings. Guest audio excluded. V1 proved feasibility, not robustness. Severe class imbalance and incomplete evaluation made generalization unknowable.
V1 Original Dataset Reality
Total Samples: 3,859 Chamath: 716 (100% train) Friedberg: 1,483 JCal: 840 Sacks: 820 Issues: • Severe class imbalance • No guest hard-negatives • No reliable generalization measurement

V2 — Guest-Aware Scaling

Same architecture. Balanced dataset. Guest awareness added. Data discipline changed.

V3 — Embedding Upgrade (NeMo / TitaNet)

Cleaner embedding manifold. Reduced over-segmentation. Test Accuracy: 96.53% Binary Host vs Other: 98.52% Primary Confusion: JCal → Chamath

V4 — Duration Constraint & Production Reality

Trained only on >5s segments. Embedding geometry improved. Production exposed confident cluster-level failures. High accuracy ≠ production reliability.

Rationale for Dropping the Classifier Layer

The failure surfaced during manual transcript spot-checking. A 10.4s segment was incorrectly marked as propagated.

Brad Gerstner’s embeddings overlap with Sacks and Friedberg. That’s why his segments were getting wrongly attributed. This is a fundamental voice-similarity problem, made worse by a classifier that’s forced to act as the source of truth inside a fixed label space. Cosine distance (mean): Brad → Sacks: 0.7774 (most similar) Brad → Friedberg: 0.8007 Two things worth noting. First, I didn’t go back and audit whether any of Brad’s segments were incorrectly labeled as Sacks. The analysis focused on the Friedberg misattribution that surfaced. Second, I don’t recall doing a deeper post-mortem on the transcript processing script itself. It’s entirely possible there was additional logic influencing propagation — for example, code-gen that unintentionally constrained the classifier to label only the four hosts, even though guests were intentionally created as a class. I also find it suspicious that there is no "custom_speaker": "brad" in any of the segment records. There’s also no "custom_speaker": "jcal" — but JCal wasn’t in that episode. The difference is that Brad’s cosine cluster sits much closer to Sacks and Friedberg than it does to JCal, which makes it unsurprising that we don’t see any JCal misattribution. If Brad’s clips were never actually eligible to be labeled as Brad at inference time, then the system wasn’t merely confused — it was structurally incapable of being correct. The classifier layer was removed. Cluster first. Verify identity. Then propagate.
