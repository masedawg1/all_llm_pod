Speaker Identification Architecture

Speaker Classification Policy Evolution

V1 — Host-Only Classifier

The first iteration of a multi-speaker classifier was created using four binary classifiers to extract about 9,000 audio files for the hosts from 98 podcast episodes. WhisperX handled ASR processing, and Pyannote embeddings drove speaker mapping. These steps produced the first 187 transcripts for the app.

Noteworthy: Guests were deliberately excluded. Guest audio was difficult to reliably gather at scale. Despite this, the system did not misclassify guests as hosts — a fifth "unknown" class was unexpectedly created during training, and low-confidence guest segments were correctly absorbed into it. Even so, the system was basic and fragile, and required a redesign.

V1 — Core Signals
V1 Summary: • ~9,000 host-only clips from 98 episodes • WhisperX segmentation + Pyannote embeddings • Guests excluded → "unknown" class emerged and correctly absorbed them • Produced first 187 transcripts; system was functional but fragile Dataset Balance (post-clean): • Sacks: 1602 • JCal: 1601 • Friedberg: 1573 • Chamath: 1810 → Nearly perfect class balance Model Suite Results: • RandomForest: CV 0.930, Val 0.972 • SVM: CV 0.936, Val 0.979 • Logistic Regression: CV 0.918, Val 0.977 • MLP: CV 0.927, Val 0.979 → Best model: MLP (97.9% validation) Test Performance: • Overall accuracy: 0.953 • Sacks F1 0.97 • JCal F1 0.96 • Friedberg F1 0.95 • Chamath F1 0.94 (Recall 0.99 → over-called boundaries) Confusion Behavior: • Main confusion: Chamath ↔ JCal (fast-speech similarity) • Sacks highly stable • Friedberg slightly overlaps but not collapsing Key Insight: • V1 proved host-only classification works • Accuracy was strong, but architecture lacked robustness • Data diversity + guest integration required a redesign

V2 — Guest-Aware Scaling

V2 was made possible via a custom HITL tool I created called Clownfish. Clownfish ingested the original V1 transcripts and let me manually review and re-map pyannote speaker_ids to structured custom_speaker labels, with side-by-side audio examples and video screengrabs for cross-validation.

Once a speaker was validated and assigned, the transcript was automatically backfilled with the corrected labels. Audio for each segment was then extracted into clean, per-speaker directories. This produced the first reliable, guest-inclusive dataset and established a disciplined labeling pipeline.

V2 — Core Signals
V2 Summary: • First HITL pipeline (Clownfish) introduced • pyannote → custom_speaker remapping validated with audio + video • Full transcript backfill applied after each speaker confirmation • Produced clean, structured per-speaker audio directories Balanced Dataset (V2-Actual): • Chamath: 2,280 total (1,810 train, 234 val, 236 test) • Friedberg: 2,280 total (1,573 train, 345 val, 362 test) • JCal: 2,280 total (1,601 train, 339 val, 340 test) • Sacks: 2,280 total (1,602 train, 348 val, 330 test) → Total V2-Actual: 9,120 samples Modeling Insight: • Class balance fully corrected • Guest segments present → realistic negative examples • Evaluation now meaningful (V1 metrics were inflated from host-only domain) Key Shift: • V2 established quality control, structure, and true dataset hygiene • Marked the transition from "feasibility" to a scalable training pipeline

V3 — Embedding Upgrade (NeMo / TitaNet)

V3 replaced Pyannote embeddings with Nvidia NeMo / TitaNet, immediately stabilizing the embedding space. Clusters became tighter, cosine distances became reliable, and label consistency improved across the entire dataset. Accuracy dipped slightly, but the geometry improved dramatically — the first version that behaved like a production-quality speaker model.

V4 — Duration Constraint & Production Reality

Whitepapers warn that sub-5s audio destabilizes embeddings, and this matched what I observed in practice.

Community advice was to drop short segments entirely — which does make sense: shorter audio is more likely to be mislabeled, and contains less acoustic identity. But roughly 50% of All-In segments are under 4 seconds, and discarding half the dataset wasn't acceptable without trying something more principled.

My working hypothesis became:
1. Train the classifier only on host and guest audio longer than 5 seconds
2. Classify only >5s segments, leaving all short segments untouched
3. After validating the long-segment classifications, propagate labels to short segments using NeMo cluster_id, which had already proven reliable

Classifier Removal — Structural Failure

Classifiers were dropped after V4 because spot-checking transcripts exposed contradictions that simply shouldn't happen:

1. Longer segments (>5s) that were directly labeled by the classifier were being marked is_back_propagated: true — which is impossible. Backfill only applies to sub-5s segments.

2. Speakers were being mislabeled, with guest content being labeled as host.

  "text": "Number one, if we flashback to what went on in China with Toutiao and Douyan, remember that text-based social networks were where it started there.",
  "speaker_id_propagated": true
}
Short Segment · Also Misclassified
{
  "speaker_id": "SPEAKER_04",
  "is_lizard": true,
  "custom_speaker": "friedberg",
  "confidence": 0.0,
  "duration": 0.3200,
  "text": "Where is Jcal?",
  "speaker_id_propagated": true
}

Brad's embedding geometry explained the second failure: his cosine distances placed him directly on top of Sacks and Friedberg in embedding space, making host/guest separation impossible for the classifier. It wasn't confused — it was incapable of being correct.

It's unclear why custom_speaker: "brad" was never created. If the labeling had been incorrectly limited to hosts, I'd expect to see his segments attributed to JCal (who wasn't present in the episode) — but that didn't happen. That bolsters the case that Brad's voice was simply too phonemically close to the hosts for the classifier to work effectively, regardless of label space.

That's why the classifier layer was removed. A forced, incomplete label space sitting on top of overlapping embeddings isn't truth — it's noise with confidence scores.

Brad — Embedding Geometry
Cosine Distances (Brad → Hosts): • Brad → Sacks: 0.7774 • Brad → Friedberg: 0.8007 Context: • Typical inter-speaker distance (V3): ~0.99 • Brad's distances fall well within host cluster overlap zones • No custom_speaker: "brad" in label space → classifier could never output Brad at inference • Result: Brad segments absorbed into nearest host cluster (Sacks / Friedberg) • Misclassification was deterministic, not stochastic
