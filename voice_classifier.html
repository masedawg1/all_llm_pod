Voice Classification Architecture

Speaker Classification Policy Evolution

The classifier evolved in three stages. V1 exposed structural fragility. V2 fixed the data. V3 upgraded the embedding backbone.

Version 1 — Host-Only Classifier

structural fragility
V1 used WhisperX segmentation and Pyannote embeddings. Four binary classifiers extracted ~9,000 host clips from 44 episodes and trained a multi-speaker model for the four hosts. Guest audio was intentionally excluded. At the time, it seemed unnecessary and operationally tedious to collect. The flaw became clear later: without guest hard-negatives, a host-only classifier risks mislabeling guest speech as a host — contaminating downstream persona training. An “unknown” class appeared and captured low-confidence segments, preventing catastrophic contamination. But this was incidental, not architectural. The system was structurally fragile.

Version 2 — Guest-Aware Scaling

data correction
V2 reused the same WhisperX + Pyannote embedding pipeline, but introduced verified speaker mapping through Clownfish and large-scale guest extraction. Dataset scale: - 61,682 clips - 198.46 hours - 4 hosts + 87 guests Embeddings: Pyannote Models: RandomForest, SVM, Logistic Regression, MLP, XGBoost Results: - Host vs Guest: ~99% accuracy - 5-class (hosts + other): ~96–97% V2 solved the structural flaw by balancing the dataset and introducing explicit guest awareness. The architecture remained the same; the data discipline changed.

Version 3 — Embedding Upgrade (NeMo / TitaNet)

representation correction
V3 replaced Pyannote embeddings with NVIDIA NeMo TitaNet embeddings. Segmentation, balancing strategy, and classification architecture remained conceptually unchanged. The representation layer improved. Results: - 4-host classifier: 99.38% test accuracy - Guest classifier (93 classes): 97.67% - Reduced confusion between similar hosts V2 fixed the dataset. V3 fixed the feature space.
