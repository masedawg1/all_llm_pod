Speaker Classification Policy Evolution

V1 — Host-Only Classifier

WhisperX segmentation + Pyannote embeddings. Guest audio excluded. V1 proved feasibility, not robustness. Severe class imbalance and incomplete evaluation made generalization unknowable.
V1 Original Dataset Reality
Total Samples: 3,859 Chamath: 716 (100% train, 0 val, 0 test) Friedberg: 1,483 (1,016 train, 226 val, 241 test) JCal: 840 (594 train, 123 val, 123 test) Sacks: 820 (581 train, 129 val, 110 test) Issues: • Severe class imbalance • Chamath had no validation/test split • No guest hard-negatives • No reliable generalization measurement

V2 — Guest-Aware Scaling

Same architecture. Balanced dataset and introduced guest awareness. Data discipline changed.
V2 Balanced Dataset
Total: 9,120 samples Each Host: 2,280 samples Chamath: 1,810 train / 234 val / 236 test Friedberg: 1,573 train / 345 val / 362 test JCal: 1,601 train / 339 val / 340 test Sacks: 1,602 train / 348 val / 330 test

V3 — Embedding Upgrade (NeMo / TitaNet)

Replaced Pyannote with NVIDIA NeMo TitaNet. Cleaner embedding manifold. Reduced over-segmentation.
V3 Performance
Embeddings: 91,410 Train / Val / Test: 64,131 / 13,062 / 14,217 Test Accuracy: 96.53% Host Identification: 95.98% Binary Host vs Other: 98.52% Combined End-to-End: 91.19% Primary Confusion: JCal → Chamath (196)

V4 — Duration Constraint & Production Reality

Trained only on >5s segments. Anchored identity on long-form geometry. Backfilled short segments via cluster mapping. Embedding geometry improved. Production exposed confident cluster-level failures. High accuracy ≠ production reliability.
V4 Evidence

Rationale for Dropping the Classifier Layer

The failure of the V4 classifier was non-obvious. It surfaced during manual transcript spot-checking for logical inconsistencies.

Expected Logic
• Segments ≥5s are directly classified → speaker_id_propagated = false • Segments <5s are backfilled from validated clusters → is_lizard = true
Instead, a 10.4s segment appeared with: speaker_id_propagated = true This should never happen under V4 rules.

What That Revealed
The long segment was labeled as Friedberg. But it was Brad Gerstner. This exposed a deeper issue: confident cluster-level mislabeling.

Brad Gerstner Embedding Analysis
327 Brad clips were compared against all hosts. Average cosine distances: • Brad → Sacks: 0.7774 • Brad → Friedberg: 0.8007 • Brad → JCal: 0.8665 • Brad → Chamath: 0.9181 Baseline: • Average intra-host distance: 0.3642 • Average inter-host distance: 0.8740 Critical finding: Minimum Brad → JCal distance: 0.2507 This is below the average intra-host baseline. Meaning: Some Brad clips are closer to JCal than hosts are to themselves.

Implication
Brad’s embedding manifold overlaps with Sacks and Friedberg. A forced 5-class classifier will confidently mislabel him. Confidence was not the issue. Label-space constraint was.

Conclusion
The mis-propagated long segment exposed the architectural flaw. High accuracy ≠ production reliability. The classifier layer was removed. Cluster first. Verify identity. Then propagate.
