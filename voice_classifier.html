Voice Classification Architecture

Speaker Classification Policy Evolution

The classifier evolved in three stages. V1 exposed structural fragility. V2 corrected the dataset. V3 upgraded the embedding backbone.

Version 1 — Host-Only Classifier

structural fragility
Pipeline
WhisperX segmentation + Pyannote embeddings.
Four binary classifiers extracted ~9,000 host clips from 44 episodes.

Design Assumption
Guest audio was excluded intentionally. At the time, it appeared unnecessary and operationally tedious.

Structural Flaw
Without guest hard-negatives, a host-only classifier risks labeling guest speech as a host — contaminating downstream persona training.

Incidental Safeguard
An “unknown” class captured low-confidence segments. This prevented catastrophic contamination — but it was incidental, not architectural.

Version 2 — Guest-Aware Scaling

data correction
V2 reused the same segmentation and embedding stack, but introduced verified speaker mapping (Clownfish) and large-scale guest extraction. The architecture remained the same. The data discipline changed.
V2 Dataset & Results
Dataset scale: - 61,682 clips - 198.46 hours - 4 hosts + 87 guests Embeddings: - Pyannote Models: - RandomForest - SVM - Logistic Regression - MLP - XGBoost Results: - Host vs Guest: ~99% accuracy - 5-class (hosts + other): ~96–97% V2 introduced explicit guest awareness and class balance.

Version 3 — Embedding Upgrade (NeMo / TitaNet)

representation correction
V3 replaced Pyannote embeddings with NVIDIA NeMo TitaNet embeddings. Segmentation logic and classification architecture remained conceptually unchanged. V2 fixed the dataset. V3 fixed the feature space.
V3 Performance
Results: - 4-host classifier: 99.38% test accuracy - Guest classifier (93 classes): 97.67% - Reduced confusion between similar hosts Improvement source: - Better embedding geometry - Unit-norm cosine stability - Stronger class separability
