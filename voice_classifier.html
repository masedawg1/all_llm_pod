Speaker Identification Architecture

Speaker Classification Policy Evolution

V1 — Host-Only Classifier

Initial feasibility experiment. WhisperX segmentation with Pyannote embeddings. Guest audio excluded from the dataset.

V1 Dataset Breakdown
Total V1: 3,859 samples Chamath: 716 Friedberg: 1,483 Jcal: 840 Sacks: 820 Issues • Severe class imbalance • Guest audio excluded • No meaningful generalization test

V2 — Guest-Aware Scaling

Dataset discipline introduced. Host classes balanced and guest audio included.

V2-Actual Balanced Dataset
Chamath: 2,280 total (1,810 train, 234 val, 236 test) Friedberg: 2,280 total (1,573 train, 345 val, 362 test) Jcal: 2,280 total (1,601 train, 339 val, 340 test) Sacks: 2,280 total (1,602 train, 348 val, 330 test) Total V2-Actual: 9,120 samples

V3 — NeMo / TitaNet Embeddings

V3 Classifier Performance
Binary classifier (host vs other): 98.52% Host identification accuracy: 96.73% End-to-end system accuracy: 91.19% Best model: Random Forest Test accuracy: 96.53% Primary confusion: Jcal → Chamath: 196 errors Chamath → Jcal: 43 errors Other classes remain above ~97–99% accuracy.

V4 — Duration Constraint (>5s)

Embedding Geometry Shift
Old separation ratio: 1.19× New separation ratio: 4.18× Embedding separation improved by roughly +250%. Accuracy decreased slightly: 98.5% → 96% But cluster structure became dramatically more stable and generalizable.
Why Accuracy Dropped
Short clips (<5s) were removed. Short segments often contain extremely strong voice cues. Removing them introduces more difficult examples: • overlapping speech • mixed acoustic environments • harder speaker boundaries The classifier therefore becomes more honest about uncertainty.

Rationale for Dropping the Classifier Layer

Brad Gerstner’s embeddings overlap with both Sacks and Friedberg. Cosine distance (mean): Brad → Sacks: 0.7774 Brad → Friedberg: 0.8007 If the inference system cannot output the label “brad,” the model is not confused — it is structurally incapable of being correct. Correct pipeline: Cluster speakers first. Verify identity externally. Then propagate speaker labels.
